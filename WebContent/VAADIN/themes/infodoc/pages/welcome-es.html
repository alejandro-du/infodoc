¡Bienvenido!

InfoDoc Pro le ofrece una aplicación web fácil de usar para ejecutar y controlar procesos de negocio tales como:

  • Gestión de Tickets o Incidentes
  • Gestión de Correspondencia (Solicitudes y Respuestas)
  • Gestión de Servicios
  • Gestión de Clientes (CRM)
  • Gestión de Tareas o Actividades
  • Gestión de Proyectos y Requerimientos
  • Gestión de Facturas
  • Gestión de Documentos

Con InfoDoc Pro puede definir formularios (casos, tickets, incidentes, solicitudes, respuestas, servicios, clientes, tareas, actividades, proyectos, requerimientos, facturas, etc.) que serán manipulados ejecutando actividades. Una actividad permite manipular las propiedades (o metadatos) incluidos en un formulario.

Primeros Pasos

1. Cree grupos y usuarios
Seleccione las opciones "Grupos" y "Usuarios" del menú "Administración" y cree todos los grupos y usuarios que requiera.
2. Cree formularios para gestionar los datos de sus procesos
Seleccione la opción "Formularios" del menú "Configuración" y defina sus formularios.
3. Incluya propiedades en sus formularios
Seleccione la opción "Propiedades" del menú "Control" y defina todas las propiedades requeridas para completar sus formularios.
4. Cree tipos de transacciones
Seleccione la opción "Actividades" del menú "Configuración" y defina los pasos de sus procesos.
5. Explore la documentación y videos tutoriales
Lea la documentación disponible en el menú "Ayuda" y explore la sección de Videos Tutoriales de esta página.

Videos Tutoriales

¿Cómo configuro InfoDoc Pro para la Gestión de Tickets o Incidentes?

Ejemplo de configuración de InfoDoc Pro para la Gestión de Tickets o Incidentes. Un Ticket es un caso de atención a un cliente que pasa por estados como Abierto, En progreso y Cerrado. Cada Ticket contiene información del cliente (compañía, usuario) y del caso (tipo, descripción, resolución, estado).

¿Cómo configuro InfoDoc Pro para la Gestión de Correspondencia (Solicitudes y Respuestas)?

Ejemplo de configuración de InfoDoc Pro para la Gestión de Correspondencia (Solicitudes y Respuestas). Una Solicitud es un documento que llega a una compañía. Una Repuesta es un documento que sale de la compañía. Es por esto que un sistema de Gestión de Solicitudes y Respuestas puede ser visto como un sistema de gestión de Correspondencia.

¿Cómo configuro InfoDoc Pro como Gestión de Clientes (CRM)?

Ejemplo de configuración de InfoDoc Pro para la Gestión de Clientes (CRM). Un CRM simple, debe permitir la gestión de Clientes, Citas y Llamadas Telefónicas. Un Cliente incluye información personal, de contacto y de negocio. Cada Cliente tiene una lista de Citas y Llamas Telefónicas.

¿Cómo configuro InfoDoc Pro para la Gestión de Proyectos y Requerimientos?

Ejemplo de configuración de InfoDoc Pro como sistema de Gestión de Proyectos y Requerimientos. Un Proyecto contiene datos como nombre, cliente, personas a cargo, acta de inicio, riesgos, prioridad, fecha de próxima entrega, fecha final y requerimientos de la próxima entrega. Un Requerimiento contiene datos como título, proyecto tipo, descripción, criterio de aceptación, esfuerzo estimado y esfuerzo restante.
